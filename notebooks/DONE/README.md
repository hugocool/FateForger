# DONE Notebooks

Human-readable index for completed notebook work.

## Purpose

- Keep finalized notebooks tied to completed issues/PRs.
- Preserve analysis/report context after extraction into modules/tests/docs.

## Definition of Done for notebooks

- Linked issue/PR is complete.
- Notebook metadata is current and points to durable artifacts.
- Notebook reruns cleanly from a fresh kernel.
- Duplicated production logic is removed or intentionally justified.
